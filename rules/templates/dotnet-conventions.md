## Code Style

- Nullable reference types: **enabled** — treat all nullability warnings as errors
- `async`/`await` for all I/O operations — no `.Result` or `.Wait()` blocking calls
- Constructor injection for all dependencies — no service locator pattern
- `CancellationToken` parameter on all async public methods
- `record` types for DTOs and value objects
- `sealed` on classes that are not designed for inheritance

## Naming

- Classes, methods, properties, constants: `PascalCase`
- Local variables, parameters: `camelCase`
- Private fields: `_camelCase`
- Interfaces: `IServiceName`
- Async methods: suffix with `Async` (e.g. `GetByIdAsync`)

## Testing

- **Framework**: [DETECTED: xUnit | MSTest | NUnit]
- Test project naming: `[ProjectName].Tests`
- Naming: `<Method>_<Scenario>_<ExpectedBehaviour>` (e.g. `GetById_ExistingId_ReturnsEntity`)
- Use `Moq` for mocking (or [DETECTED: NSubstitute | FakeItEasy])
- Use `FluentAssertions` for readable assertions (if detected)
- Structure: Arrange / Act / Assert with blank lines separating sections
- Unit tests: mock all external dependencies
- Integration tests: use `WebApplicationFactory<T>` for API layer (if ASP.NET detected)

```csharp
[Fact]
public async Task GetById_ExistingId_ReturnsEntity()
{
    // Arrange
    var repo = new Mock<IEntityRepository>();
    repo.Setup(r => r.GetByIdAsync(It.IsAny<Guid>(), It.IsAny<CancellationToken>()))
        .ReturnsAsync(TestData.CreateEntity());
    var service = new EntityService(repo.Object);

    // Act
    var result = await service.GetByIdAsync(Guid.NewGuid());

    // Assert
    result.Should().NotBeNull();
    result!.Id.Should().NotBeEmpty();
}
```

## Anti-Patterns

- No `.Result` or `.Wait()` on async methods — always `await`
- No `catch (Exception ex)` without re-throwing or specific handling
- No `static` mutable state
- No `public` fields — use properties
- No nullable-oblivious code — resolve all nullable warnings
