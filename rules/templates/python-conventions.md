## Testing

- **Framework**: [DETECTED: pytest | unittest]
- Test files: `tests/` directory, mirror the `src/` structure
- Naming: `test_<module>_<scenario>_<expected>`
- Structure: Arrange / Act / Assert
- Async tests: `@pytest.mark.asyncio` (if asyncio detected)
- Mock external services — do not call live APIs or databases in unit tests
- Use `pytest-httpx` or `respx` for HTTP mocking (if detected)
- Use `moto` for AWS service mocking (if boto3 detected)

## Async

- Use `async def` / `await` for all I/O operations
- Do not call `asyncio.run()` inside async functions
- Use `asyncio.gather()` for concurrent operations

## Dependencies

- Add via: [DETECTED: `uv add <package>` | `poetry add <package>` | `pip install <package>`]
- Pin versions in `pyproject.toml` or `requirements.txt`
- Separate dev dependencies from production dependencies

## Anti-Patterns

- No `from module import *`
- No mutable default arguments (`def f(x=[])` -> use `None`)
- No bare `except:` — always catch specific exceptions
- No `print()` in production code — use `logging`
- No blocking I/O in async functions
